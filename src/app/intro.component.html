This Angular application shows how the density theming configuration affects the Angular Material components.

Problem #1 -3 global density crashes compilation

GitHub issue 26613

Currently (@angular/material v15.2.0), the compiler throws the following error when setting density globally "@include mat.all-component-densities(-3);" 

      Module build failed (from ./node_modules/sass-loader/dist/cjs.js):
      SassError: "mdc-density: height must be between 24px and 32px (inclusive), but received 20px."
      ╷
      632 │     $height: density.prop-value(

This should be possible based on Material documentation about Density scale: "The density scale is numbered, starting at 0 for a component’s default density. The scale moves to negative numbers (-1, -2, -3) as space decreases,..."

The mat-chips implementation produces this error.

Problem #2 mat-label disappears when density < -1

GitHub issue 26579

The mat-label disappears when the density of mat-form-field with appearance="fill" is set to lower than -1.
